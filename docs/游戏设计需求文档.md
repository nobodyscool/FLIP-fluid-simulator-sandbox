# 像素调酒模拟经营游戏 — 设计需求文档

> 版本：draft v0.1　　用途：demo开发范围界定与团队/自查参考

---

## 一、游戏概述

玩家扮演一名穿越不同时空的调酒师，在各个时代/地点开设酒吧，用拟真的调酒流程为形形色色的顾客调制鸡尾酒，积累资金并解锁新工具、新配方、新玩法。

**核心卖点**：
- 聚焦"调酒"本身的沉浸式操作体验，玩法对标 *VA-11 Hall-A*（红弦俱乐部），但**视觉表现力显著更强**是本作差异化重点。
- 2D 像素美术风格，但不同关卡（不同时空背景）画风各异，避免视觉单调。
- 调酒过程高度还原真实调酒的步骤、工具和原料。

**类型定位**：模拟经营 + 视觉小说式沉浸互动，弱数值成长，强操作/观感体验。

## 二、核心玩法循环

1. **备货阶段**：营业前自设酒单（决定当天能调制哪些鸡尾酒），并自行采购/准备对应原料。
2. **营业阶段**：顾客陆续上门，根据顾客需求或点单，实际操作调酒流程完成制作。
3. **反馈阶段**：顾客根据成品品质给出评价与报酬。
4. **结算与解锁**：用赚取的资金购买新工具、新基酒、新原材料、新的鸡尾酒说明书（配方书），解锁新的玩法可能性。
5. 循环回到备货阶段，进入下一天/下一关卡（可能是新的时空场景）。

**设计意图**：通过"营业前自设酒单 + 自备原料"这两条限制，天然控制单日调酒的复杂度上限，避免规则膨胀导致的操作过载，同时创造"策略性备货"的决策深度。



## 三、世界观与关卡结构

- 玩家角色是一名在不同时空旅行的调酒师，每个关卡代表一个不同的时代/地点场景。
- 不同关卡在美术风格上应有明显区隔（例如像素分辨率、调色板策略、光影处理都可不同），但整体保持"2D像素"的统一媒介语言。
- 每个新关卡对应新的顾客群体、新的可用工具/原料、以及相应难度提升的鸡尾酒配方。

**待定事项**：
- 具体时空场景设定（几个关卡、分别是什么时代/地点）尚未定案。
- 关卡间的解锁顺序是线性还是有分支，待设计。



## 四、调酒系统设计

### 4.1 拟真调酒原则
- 使用**真实存在**的调酒步骤、工具和原料，不采用抽象化数值配方。
- 玩家需要实际执行倒酒、摇酒、搅拌、兑入、装饰等操作，而非选择菜单式"确认调酒"。

### 4.2 难度曲线
- 鸡尾酒难度需要**循序渐进**，随关卡/进度逐步引入更复杂的技法、更多种类的基酒与辅料。
- 前期关卡应聚焦少数几种基础操作（如简单兑和），后期逐步引入摇酒、分层、精细配比等进阶技法。

### 4.3 单日复杂度限制机制
- **营业前自设酒单**：玩家在开业前从已解锁的配方中选择当天要提供的鸡尾酒种类，非当日酒单上的酒不可制作。
- **自备原料**：玩家需要在备货阶段采购/准备好当日酒单所需的原料，原料不足则无法完成对应配方。
- 这两条限制共同作用，把"总配方库的复杂度"和"单次营业的操作复杂度"解耦，是控制游戏认知负荷的核心设计手段。

### 4.4 鸡尾酒判定系统（设计需求，技术方案见另一文档）
- 系统需要根据**杯中实际材料**（种类、用量）和**玩家的操作过程**（技法、顺序）综合判定：
  - 调出的是哪一款鸡尾酒（或判定为"未知创意配方"）；
  - 该杯成品的品质等级（如完美/合格/勉强/失败）。
- 顾客需根据判定结果给出相应的评价反馈（满意度、报酬、可能的对话/剧情反应）。



## 五、经营与解锁系统

- 通过营业赚取资金。
- 资金可用于购买：
  - **工具**（调酒器具，可能影响可执行的技法或效率）；
  - **基酒**（解锁新的酒水种类）；
  - **原材料**（辅料、装饰物、冰、糖等消耗品）；
  - **鸡尾酒说明书**（解锁新配方，是难度曲线推进的主要途径）。
- 解锁内容应与"难度循序渐进"原则联动：新工具/新配方应逐步引入新的操作维度，而非一次性开放全部复杂度。

**待定事项**：
- 具体的经济数值曲线（定价、收入、成本）尚未设计。
- 是否存在关卡间可携带的"存档式"进度，还是每关独立重置，待定。



## 六、顾客与反馈系统

- 顾客是游戏叙事与反馈的主要载体，类似 *VA-11 Hall-A* 的顾客对话/点单模式。
- 顾客可能：
  - 直接点单指定鸡尾酒名称；
  - 或仅描述需求/口味偏好，由玩家judgment调制；
- 顾客对成品的反馈应体现在：评分/满意度、报酬、可能的对话内容变化或剧情推进。

**待定事项**：
- 顾客是否有独立的人物剧情线（类似VA-11 Hall-A的角色支线），待设计。
- 顾客点单的具体交互形式（文本对话/语音/图标提示）待定。



## 七、视觉呈现需求（面向玩家体验，非技术实现）

游戏要求2D酒杯/酒瓶中的液体和固体呈现具备"可信的物理感"，具体视觉体验需求至少如下：

1. **基酒瓶液面可见**：透明酒瓶中的基酒剩余量，应能通过液面高度直接观察到。
2. **倾倒物理与视觉效果**：倒入液体时应有可信的"体积感"（不要求真实流体模拟精度），且**基酒瓶倾斜角度不同时，水流大小应有相应变化**（角度越接近水平，流速/水流越大）。
3. **冰块投入的动态效果**：冰块落入杯中需要有真实感的下落轨迹，并在**接触液面时产生冲击反馈与涟漪效果**。
4. **杯中固体物质的合理呈现**：水果、冰块、糖粒、吸管等固体物，需要有合理的物理表现（如浮动、轻微晃动）和视觉细节，不要求完全真实物理模拟。
5. **杯子可被鼠标拖动摆放**：玩家可以将杯子拖动到桌面任意位置；拖动过程中，杯内液面应根据拖动产生的加速度左右起伏，杯中固体也应相应晃动，呈现"端着杯子移动"的可信手感。
6. **（可选）超现实粒子/光效**：用于增强特定鸡尾酒或关键操作的表现力，非强制项，视demo开发进度决定是否加入。

**视觉风格取向**：整体采用像素美术（液体主体的色彩/形状/波纹保持像素化调色板与硬边过渡处理），但涟漪、辉光、粒子等辅助特效可适度采用非像素化的平滑渲染，形成"像素主体 + 精致光效点缀"的现代混搭视觉语言（参考 *Hyper Light Drifter*、*Dead Cells* 一类处理方式）。
