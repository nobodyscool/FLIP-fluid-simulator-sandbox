## 0. 对现有需求文档的总评

先说结论：**这份 draft 的地基是对的，而且有三个判断非常值钱**——

1. **"拟真操作而非菜单确认"**。VA-11 Hall-A 的调酒是刻意机械化的（选原料、按混合），因为它的灵魂在对话。我们把它舍弃的那个维度捡起来做成本体，这是成立的差异化，而不是像素酒吧题材的同质化竞争。
2. **"营业前自设酒单 + 自备原料"的复杂度节流阀**。这是全文档最聪明的一条设计。它把"配方库总复杂度"和"单夜操作复杂度"解耦，同时白送了一层策略决策。后文第 5 章会把它扩展成完整的"预报—备货"系统。
3. **"像素主体 + 平滑特效点缀"** 的视觉语言判断准确，且与本仓库已有的流体渲染路线天然吻合。

同时有三个明显的缺口：

1. **时空设定目前只是换皮**。文档把"不同时代"当作美术风格轮换的借口，但它可以是玩法本体——鸡尾酒的真实历史本身就是一条从简到繁的教学曲线（见第 1 章，这是本提案最核心的主张）。
2. **全文没有一个字提到声音**。拟真操作游戏的"手感"一半在音频：冰块撞杯壁、倒酒的咕咚声、摇壶节奏、气泡嘶声。没有声音的倒酒原型会显得比实际水平差很多，既影响试玩评价也影响团队士气。
3. **没有失败与混乱的空间**。打翻、溢杯、碎杯、喷发——混乱是喜剧，喜剧是切片传播（streamability）。一个只有"做对/做错"评级、没有"搞砸得很好笑"的拟真操作游戏，等于放弃了最便宜的营销渠道。

一个需要立刻警惕的风险：**"每关不同画风"是美术量炸弹**。像素美术看着省、做起来贵，每个时代 = 全套场景 + 顾客立绘 + 酒瓶道具重绘。demo 阶段必须单场景做深（见第 9 章）。

> 值得聊：对标产品除了 VA-11 Hall-A，建议团队集体过一遍这四款——Cooking Simulator（拟真操作的喜剧性从哪来）、Overcooked（硬时间压力的代价）、PowerWash Simulator（零压力满足感为什么成立）、Coffee Talk（叙事节奏怎么和制作节奏咬合）。我们要在"压力谱系"上选一个明确位置，见第 4 章。

## 1. 核心创意主张：鸡尾酒的真实历史，就是天然的难度曲线

调酒术不是一堆平行配方，它是**线性演化**出来的：

- 1806 年报纸上 "cock-tail" 的原始定义 = 烈酒 + 糖 + 水 + 苦精——就是一杯 Old Fashioned。
- 镀金时代（1860s–1900s）冰贸易兴起，诞生搅拌类（Martinez、Manhattan、Martini）。
- **禁酒令（1920–1933）的私酿劣酒难以入口，才催生了用柠檬汁、蜂蜜、果汁掩盖酒味的摇和类**——Shake 家族的爆发是有真实历史原因的。调酒师逃亡欧洲，顺手把 Sidecar、White Lady 留在了巴黎和伦敦。
- 战后 Tiki 浪潮（1934–1960s）：碎冰、多朗姆、分层渐变、装饰狂欢。
- 1980s 派对黑暗时代：分层 shot（B-52）、Long Island、花式抛瓶（flair）。
- 2000s 精酿复兴：量酒器精准主义、大方冰、蛋白泡沫、日式硬摇。
- 2010s+ 分子调酒：泡沫、烟雾、球化、变色。

**所以"穿越时空"不该是皮，而是把教学曲线直接交给历史。** 每个时代解锁的技法、工具、原料，就是那个时代真实拥有的东西；难度递进不需要人工编排，历史已经排好了。

### 关卡框架（完整版愿景：6 个时代）

| # | 年代 / 场景 | 新解锁技法 | 代表酒 | 时代事件 / 演出 |
|---|---|---|---|---|
| L1 | 1862 美国西部沙龙 | 直调、杯内搅拌、滑杯上酒 | Old Fashioned、Mint Julep、Whiskey Highball | **滑杯**：沿吧台滑酒杯给远处客人（纯物理小游戏）；关底演出 = Blue Blazer 火焰抛接（调酒祖师爷 Jerry Thomas 的真实成名技） |
| L2 | 1926 禁酒令地下酒吧 | 摇和、双重过滤、涮杯 | Sidecar、White Lady、Bee's Knees、French 75 | **警察突袭**：限时把酒瓶藏进暗格（物理演出）；蜂蜜掩盖劣质金酒的历史梗直接写进对话 |
| L3 | 1954 加勒比 Tiki 酒吧 | 碎冰、搅拌机、密度分层、火焰装饰 | Mai Tai、Piña Colada、Tequila Sunrise、Zombie | Zombie 每人限两杯（Don the Beachcomber 的真实店规）；日落渐变分层 = 流体引擎炫技位 |
| L4 | 1985 霓虹迪斯科 | 分层 shot、花式抛瓶、拒卖判断 | B-52、Long Island、Blue Lagoon、Sex on the Beach | flair 抛瓶接杯（物理喜剧）；醉客管理机制引入 |
| L5 | 2012 精酿复兴 speakeasy | 精准量酒（容差收紧）、大方冰、蛋白干摇 | Espresso Martini、Penicillin、Ramos Gin Fizz | Ramos 长摇体力秀名场面；日式硬摇文化 |
| L6 | 2077 分子 / 赛博酒吧 | 泡沫、烟雾、变色、球化 | 原创未来酒 + 蝶豆花变色系 | 流体模拟的最终炫技关；剧情收束：酒馆为何旅行 |

> 值得聊：6 关是 1.0 愿景，不是承诺。EA 版本建议 3 关（L1 / L2 / L5），demo 只做一关（第 9 章）。
> 值得聊："技法即时代精神"是可传播的知识点——"摇和类是禁酒令逼出来的""Zombie 限购两杯是真店规"，这类典故做进顾客对话，玩家会主动截图讲故事。知识性内容是白嫖的口碑传播。
> 值得聊：关卡间进度建议"工具与技法永久保留、原料库存每关重置"——手艺跟人走，货跟店走，直觉上也说得通。这回应了原文档"存档式进度还是每关重置"的待定项：答案可以是"一半一半"。

## 2. 操作动词系统（本作真正的主角）

拟真调酒游戏的核心资产不是配方数量，是**操作动词的手感质量**。每个动词必须满足：新手能完成、熟手有余裕、失败有喜剧形态。

| 动词 | 输入方案（鼠标优先） | 技巧表达 | 物理 / 视觉反馈 | 失败形态 |
|---|---|---|---|---|
| **倒** | 按住拖到杯口上方，拖动角度控制倾角 | 自由倒（数节拍估量）vs 用量酒器（慢但准） | 倾角越平流量越大（原需求 §7.2）；瓶内液面实时下降 | 倒过量、洒到台面 |
| **摇** | 抓住摇壶做往复甩动 | 时长 ↔ 稀释度与温度；节奏稳定度 ↔ 泡沫质量 | 壶身结霜表示温度到位；开壶时雾气 | 没盖紧喷一身；碳酸液体入壶 = 爆喷（真实！） |
| **搅** | 在调酒杯内画圆 | 圆的平滑度与速度；过久 = 过度稀释 | 漩涡带动冰块公转 | 搅太猛溅出、碎冰浑浊 |
| **捣** | 在杯内按压节奏点击 | 力度分寸——薄荷捣烂出苦味（真实细节） | 薄荷叶碎裂、香气粒子 | 捣穿吸管 / 压碎杯底 |
| **分层** | 沿吧勺缓慢倒入 | 流速低于阈值才不穿层；需懂密度顺序 | 清晰的密度分界面（引擎已支持变密度投影） | 一手抖，层毁了 |
| **兑顶** | 沿杯壁缓倒碳酸 | 控泡——倒太快气泡溢出 | 气泡上浮、泡沫层 | 汽水喷溢一桌 |
| **加冰** | 选冰型（方冰 / 大方冰 / 碎冰）投入 | 冰型决定稀释速率与观感 | 落水冲击与涟漪（原需求 §7.3） | 砸溅、加错冰型 |
| **过滤** | 扣滤网倾壶 | 双重过滤多一步细滤 | 果肉 / 碎冰被拦下 | 滤网没扣稳，固体全进杯 |
| **削皮 / 装饰** | 小刀刮皮、摆放装饰 | 皮卷扭油：对准杯口挤压果皮 | 皮油喷雾粒子 + 光斑（超现实光效位，原需求 §7.6） | 装饰掉进酒里 |
| **火** | 火柴 / 喷枪点燃 | 燃焰橙皮、火焰漂浮层 | 火焰粒子 + 局部光照 | 烧焦装饰（喜剧，无伤害） |
| **滑杯上酒** | 沿吧台方向甩出杯子 | 力度拿捏——停在客人面前 | 杯内液面随加速度晃动（原需求 §7.5 的延伸） | 滑过头摔下吧台 |
| **擦台 / 洗杯** | 打烊环节按住擦拭 | 无压力，纯满足感 | 污渍消除、杯子发亮 | 无（禅意环节不设失败） |

**三条设计原则：**

1. **流体本身就是 UI**：颜色 = 配比，浊度 = 稀释度，泡沫 = 摇酒质量，壶身结霜 = 温度。能用模拟表达的信息，不要用进度条表达。这是"视觉表现力显著更强"这一卖点的真正兑现方式——不是更好看，是**更可读**。
2. **容差分级，不做毫升级惩罚**：判定按区间带（完美 / 合格 / 勉强 / 失败），量酒器让区间带更容易命中，自由倒靠肌肉记忆但更快、更帅。
3. **每个动词都要有失败形态，且失败要好笑**（详见第 8 章）。

> 值得聊：摇酒输入有真实分歧——方案 A 鼠标真实甩动（累、拟真、直播效果炸裂）vs 方案 B 节奏点击（可控、抽象、不累手）。建议 A 为主体验、B 作为可及性选项。这类"体感投入"正是与 VA-11 按钮式调酒拉开差距的地方，不宜为省事全盘退回 B。
> 值得聊："滑杯上酒"是我强烈推荐新增的动词——西部沙龙的标志性动作、纯物理玩法、天然的 GIF 素材，且直接复用现有的杯体拖拽 + 液面晃动系统，开发性价比极高。

## 3. 判定系统：配方匹配 + 风味向量

原需求 §4.4 要求"根据杯中实际材料与操作过程判定"。展开成两层：

### 3.1 指名点单（"来杯玛格丽特"）
- **配方模板匹配**：材料集合落在容差带内 && 必要技法已执行（该摇的摇了、该分层的分了）。
- **品质分** = 配比精度 × 稀释度（摇/搅时长）× 温度 × 杯型与装饰 × 整洁度（有没有洒）。
- 输出等级：完美 / 合格 / 勉强 / 失败（沿用原文档四档）。

### 3.2 模糊点单（"来点苦的、烈的，像我前任一样"）
- 建立**风味向量空间**：每种原料贡献一个向量 `[甜, 酸, 苦, 烈, 果香, 草本, 气泡, 奶感, 温度]`，成品 = 按用量加权求和 + 技法修正（摇 = 降温增稀释，蛋白 = 增奶感……）。
- 顾客需求 = 目标区域；成品到区域的距离 → 评价档位。
- **创作命名机制**：判定为"未知配方"且顾客满意时，顾客为这杯酒起名，它进入你的**招牌菜单**，后续顾客可能指名点它。玩家的创作被世界记住——这是拟真沙盒最动人的回报。

> 值得聊：判定透明度是差评保险。上酒后给一张"顾客口味雷达 vs 你的成品"复盘小图，把玄学变成可学习系统。模糊点单如果不可学习，必被骂"评价随机"。
> 值得聊：风味向量表其实就是《调酒师手册》第七章原料总表的数值化版本，两份文档应由同一人维护，避免手册说酸、数值表说甜。

## 4. 节奏与压力设计：明确选择"软时间压力"

**立场：操作本身已经消耗灵巧度，时间再上硬压力 = 双重惩罚 = 挫败感。**

- 顾客耐心只影响**小费系数与对话温度**，不判失败；等太久的顾客会离开（损失收入），但夜晚不会"Game Over"。
- 营业以**时钟**结束而非清空队列——没做完的单子退款了事。
- 无障碍模式：冻结耐心衰减。
- 游戏的"混乱感"来自物理（溅洒、手抖、分层失败），不来自倒计时。这是与 Overcooked 的本质分野，也更贴合 VA-11 / Coffee Talk 受众的预期。

> 值得聊：要不要给爽玩人群一个可选的"高峰挑战夜"（hard mode）？周挑战 / 排行榜都放在这个模式里，不污染主线的松弛感。低成本满足两类人。

## 5. 单日循环细化（扩展原文档第二章）

```
┌─ 备货阶段 ──────────────────────────────┐
│ 1. 看预报：报纸/电台透露今晚客群构成      │
│    （码头工人夜→威士忌直调走量；          │
│      名流派对→摇和甜酒；有酒评人暗访…）    │
│ 2. 定酒单：从已解锁配方中选 4–6 个上架槽   │
│ 3. 采购：烈酒不过期 / 鲜果香草 1–2 天损耗  │
│    / 冰按日消耗 / 蛋奶当日有效             │
├─ 营业阶段 ──────────────────────────────┤
│ 点单 → 制作（操作动词系统）→ 上酒         │
│ （递送/滑杯/端杯拖拽=晃洒风险）→ 反馈      │
├─ 打烊阶段 ──────────────────────────────┤
│ 收入/损耗/小费复盘 → 剧情结算 →           │
│ 擦台洗杯（无压力满足感收尾）→ 存档         │
└─────────────────────────────────────────┘
```

- **酒单槽位数**就是全局难度旋钮：前期 3 槽，后期 6 槽。原文档"复杂度节流阀"的具体化。
- **原料损耗**让备货有真实张力：多买怕烂、少买怕断货。断货时可以现场对顾客说"这款没了"——顾客的反应本身是内容。
- 预报只给**部分信息**（"听说今晚有位记者……"），预测准确度可以通过升级改善。

> 值得聊：预报准确率做成可升级链（买报纸 → 装电台 → 买通线人），把"策略备货"从静态规则升级成成长系统。这是经营层的主要深度来源，成本却很低。
> 值得聊：打烊擦台是刻意设计的"呼吸拍"——PowerWash 式零压力满足感，同时顺手承载工具维护（铜杯抛光、摇壶清洗）。一夜的紧张操作需要一个禅意出口再进结算，情绪曲线才完整。

## 6. 顾客与叙事：选集结构 + 三条金线

**结构主张：每个时代是一部独立短篇集（anthology），全程只埋三条贯穿线。**

1. **神秘常客"旅人"**：每个时代都出现的同一张脸。不变老？还是另一个穿越者？他/她每次只点同一杯酒——这杯酒随时代变迁的呈现方式本身就是叙事。
2. **酒馆为何旅行**：世界观核心谜题，L6 收束。酒馆本身是活的/是一件时间装置/是你的心象——留待定案。
3. **配方的传承痕迹**：你在 L1 发明的创作酒，可能在 L5 的酒谱书里被"考古"出来——你自己就是典故。

**顾客系统：**
- 原型库（话痨 / 赶时间的 / 行家 / 伤心人 / 便衣警察 / 酒评人……）× 时代皮肤，复用行为逻辑、只换立绘与文案。
- **配方可以由顾客赠与解锁**，不只在商店买——老水手教你 Navy Grog，比商店购买多一层情感记忆。回应原文档"顾客剧情线深度"的待定项：不做 VA-11 那么重的个人线，做"每人一段小传 + 一次赠与时刻"的轻量版。
- **拒卖机制**（L4 引入）：客人喝多了，续杯还是递水？调酒师的真实职业义务，天然的道德小抉择，几乎零系统成本。

> 值得聊：VA-11 的灵魂是"倾听者幻想"，但它的玩家在打字机节奏里读文本；我们的玩家手上在忙。**建议对话密度与制作状态联动**：制作中顾客只说短句 / 环境音，上酒后才展开长对话。这个节奏问题建议 demo 阶段就做 AB 测试，它决定叙事组的写法。

## 7. 视觉与听觉

### 视觉
- **流体即信息**（第 2 章原则的重申，这是渲染组与设计组的共同纲领）。
- **"每杯酒都是一张 GIF"**：每款配方立项时先回答"这杯酒的可截图瞬间是什么"——答不出来就换一款。营销素材在设计期就内建，不靠后期补拍。
- 像素主体 + 平滑特效的混搭语言维持原文档判断（参考 Dead Cells / Hyper Light Drifter），补充一条：**特效的克制是格调**——辉光和粒子只给关键时刻（皮油、火焰、完美判定），日常操作保持素净，对比才成立。
- **控美术量的结构性方案**：所有时代共享同一套酒馆空间骨架（吧台位置、货架网格、门窗锚点一致），每时代只换调色板、光照方案与少量"英雄道具"（L1 的鹿角、L4 的霓虹灯管）。风格差异靠色彩与光影，不靠重建场景。

### 听觉（原文档缺失章节）
- **ASMR 层**：冰撞杯壁、倒酒咕咚声随流量变调、气泡嘶声、摇壶节拍、削皮沙沙、杯底磕台。这些声音就是"手感"的另一半。
- **时代配乐**：ragtime → 爵士 → exotica → synthwave → lo-fi → ambient，配乐即时代身份。
- **环境层**：雨夜、门铃、开门时街声涌入又隔绝——酒馆作为"庇护所"的听觉表达。

### 内建传播
- 拍照模式 + 成品慢镜回放（上酒瞬间自动录 3 秒）+ 配方分享卡（可导出图片，含配方与你的评级）。

> 值得聊：音频必须尽早进原型，哪怕先用占位音效。无声的流体演示会被系统性低估，这会影响内部决策和对外试玩的判断质量。
> 值得聊：分享卡上放不放二维码 / 商店链接？放了像营销，不放浪费。建议放"酒馆名 + 玩家起的酒名"，让内容自己长腿。

## 8. 失败即内容（原文档缺失章节）

**原则：失败要好笑、低惩罚、不打断夜晚。**

- 打翻酒液在台面持续存在 → 需要擦（擦台动词复用）；
- 溢杯、碳酸入摇壶爆喷（真实物理常识！）、火焰烧焦装饰（无伤害的喜剧级）；
- 碎杯：赔杯钱 + 扫碎片，顾客围观起哄；
- 上错酒：顾客的反应对话是专门写的内容，不是通用报错——"我点的是马天尼，但这杯……其实也不难喝"；
- 醉客升级事件（L4 起）：唱歌、搭讪、打翻别人的酒——酒馆生态的一部分。
- 重做的代价 = 原料损耗，与备货系统形成呼应（浪费有感知，但不毁灭）。

> 值得聊：直播与短视频传播主要来自这一章。物理参数不要调得太"乖"——可控的混乱是资产。建议专门留一个"物理喜剧"验收项：每个动词的失败形态是否值得截一张 GIF。

## 9. Demo 范围建议（可直接执行）

**单场景**（推荐 L2 禁酒令地下酒吧——氛围最浓、有"警察突袭"这个演出钩子；若求稳则做中性的"序章酒馆"），**5 款酒 = 5 个技术展示位**，每款对应 [DESIGN.md](DESIGN.md) 中已有或规划中的一个模拟能力：

| # | 酒款 | 展示的技术系统 | 对应引擎能力 |
|---|---|---|---|
| 1 | Old Fashioned | 大方冰刚体 + 搅拌漩涡 + 皮油粒子 | 可移动固体（冰块浮沉已支持） |
| 2 | Tequila Sunrise | 红石榴糖浆下沉、密度渐变分层 | 变密度投影（水/糖浆密度差已支持） |
| 3 | Mojito | 薄荷叶与青柠角固体浮动 + 碎冰 + 气泡兑顶 | 可移动固体 + 多相液体 |
| 4 | Whiskey Sour（蛋白版） | 摇和 + 泡沫双相 | 需新增：泡沫/两相表现 |

覆盖刚体 / 密度 / 颗粒 / 泡沫 / 分层 / 火六大表现系统，**每杯都是独立 GIF 素材**，demo 本身就是宣传片素材库。

> 值得聊：demo 酒单必须和 DESIGN.md 的功能清单对表立项——每个渲染 feature 至少被一款酒使用；没有酒使用的 feature 砍出 demo 范围。反过来，Whiskey Sour 的泡沫是目前引擎没有的，要么排期要么换酒（备选：Cuba Libre 展示碳酸气泡）。

## 10. 风险清单与对策

| 风险 | 对策 |
|---|---|
| 美术量炸弹（时代 × 画风） | 共享空间骨架 + 调色板/光照差异化；demo 单场景 |
| 操作难 + 时间压 = 双重惩罚 | 软耐心（第 4 章）；无障碍选项 |
| 拟真 vs 可玩的冲突 | 容差带判定，不做毫升级惩罚；量酒器作为"简单模式道具" |
| 模糊点单被骂"评价玄学" | 复盘雷达图，判定可学习（第 3 章） |
| 对话与操作抢注意力 | 对话密度与制作状态联动（第 6 章），demo 期 AB 测试 |
| 配方内容成本（每款要杯型/装饰美术） | 配方数预算制：demo 5 / EA 25 / 1.0 约 55（与《调酒师手册》配方库对齐） |
| 鼠标甩动摇酒累手 | 输入方案 B（节奏点击）作为可及性兜底 |

平台策略：PC / Steam 鼠标优先；手柄陀螺仪摇酒（Steam Deck 体感）列为后续加分项，不进 demo。

> 值得聊：游戏名。工作代号提案**《时代精神酒馆》（Bar Zeitgeist）**——Zeitgeist = 时代精神；英文 spirits 双关"烈酒 / 精神 / 幽灵"，一间穿越时代、供应 spirits 的酒馆，题眼全占。备选：《千杯纪行》《Last Call：打烊之前》。名字建议在 demo 前定，所有传播素材都挂在它上面。
