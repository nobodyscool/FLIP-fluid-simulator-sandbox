## 一、背景与目标

本demo要验证的核心技术是：基于GPU ComputeShader实现的FLIP/PIC混合流体求解器，用于后续项目评估该方案在实时交互场景下的可行性、性能表现与操作手感。这是一个纯技术沙盒（类似经典的"流体画笔"演示程序），玩法是让用户用鼠标像画笔一样在场景里绘制不同物质（真空/固体/水/气体），实时观察流体在重力和压力作用下的物理行为。

---

## 二、技术栈与核心算法要求

- 引擎版本：Godot 4.6
- 计算方式：ComputeShader（Godot `RenderingDevice` Compute Pipeline，GLSL Compute Shader），不使用CPU侧模拟
- 核心解算方法：
  - FLIP/PIC 粒子与网格混合（blending）：每步先将粒子速度传递到网格（P2G），网格上求解不可压缩性后，按FLIP/PIC混合比例（可调参数，如FLIP占主导、PIC做少量数值稳定化）传回粒子（G2P）
  - MAC（交错网格）：速度分量X/Y分别存储在网格cell的对应面上，压力/密度/类型存储在cell中心，遵循标准Harlow-Welch交错网格布局，避免压力求解产生棋盘状数值震荡
  - Ghost-Fluid 密度映射：本demo为单相流（只有水被主动模拟为携带质量的流体）(由于后续可能会拓展到多相流模拟，但先实现单相，实现后考虑添加多相的约束和优化，所以技术上要先做好拓展空间)，但水与空气的交界面（自由液面）需要用Ghost-Fluid方法正确处理边界压力条件（自由液面处压力取大气压参考值`p_atm`），保证液面边界不因数值扩散而模糊
  - 压力求解：使用 Jacobi迭代法 求解不可压缩性对应的压力泊松方程，迭代次数为可调参数（默认60次/物理子步），选择Jacobi是因为其并行结构规整、易于在ComputeShader中实现，此为demo阶段的合理工程取舍
  - 子步进（substepping）：每渲染帧执行固定数量的物理子步（默认4步/帧），子步长度建议锁定为固定值而不是直接用波动的帧间dt，保证不同帧率下物理表现一致
  - 暂不实现粘度：本demo不求解显式粘性扩散项，依赖数值方法自身的耗散提供近似阻尼效果，这是明确的范围裁剪，不要额外实现

## 三、窗口与网格规格

- 窗口分辨率：1600 × 1200
- 模拟网格：160 × 120（因此每个网格cell对应屏幕上 10×10 像素）
- 渲染方式：方格像素显示，每个网格cell在屏幕上显示为一个10×10像素的实色色块，同一个cell内颜色统一、无渐变过渡（这是明确的视觉要求，不要对相邻cell之间做颜色插值/平滑处理，保持清晰的网格化像素观感）

## 四、物理场数据模型（每个网格cell需要记录的字段）

| 字段 | 说明 |
|---|---|
| 类型（Type） | 枚举值，4种状态：`真空`(渲染色：黑色)、`固体`(渲染色：深灰色)、`水`(渲染色：蓝色)、`气`(渲染色：灰色) |
| 速度X（VelocityX） | 遵循MAC交错网格布局，存储在cell的左右面上 |
| 速度Y（VelocityY） | 遵循MAC交错网格布局，存储在cell的上下面上 |
| 压力（Pressure） | 存储在cell中心，由Jacobi迭代求解得出；自由液面（水/气交界）处的边界值参考`p_atm` |
| 密度（Density） | 存储在cell中心，本demo为单相流，密度场主要用于Ghost-Fluid边界处理与数值稳定性，不需要处理多相密度差异 |

请设计合理的GPU侧数据布局（如多张纹理分别存储不同字段，或打包进纹理的不同通道），并确保CPU（GDScript）与GPU（compute shader）之间必要的数据交互（如鼠标画笔输入、debug信息读回）能正确同步。

## 五、交互与控制需求

| 操作 | 行为 |
|---|---|
| 按住鼠标左键拖动 | 用当前选中的画笔物质在鼠标经过的区域实时显示绘制预览（不直接写入仿真场，仅做视觉预览层）|
| 松开鼠标左键 | 将本次拖拽绘制的区域正式写入仿真场（提交生效，之后开始参与物理演算）|
| 空格键（Space） | 暂停/恢复仿真（暂停时画笔预览与绘制提交仍可进行，但物理步进冻结）|
| 数字键 `1` | 切换画笔物质为 真空 |
| 数字键 `2` | 切换画笔物质为 固体 |
| 数字键 `3` | 切换画笔物质为 水 |
| 数字键 `4` | 切换画笔物质为 气体 |
| 数字键 `0` | 切换画笔为 压力编辑模式（此时鼠标绘制操作直接对cell的压力场赋值/施加扰动，而非改变物质类型，用于测试压力扰动来源）|
| 鼠标滚轮 | 增大/减小画笔半径，单位为网格数（例如滚轮向上一格，画笔半径增加1个cell），需要在屏幕上有画笔半径的可视化指示（如绘制一个跟随鼠标的圆形/方形轮廓）|
| `P` 键 | 清空整个场景的速度场（所有cell的VelocityX/VelocityY归零，其余字段不变）|
| `V` 键 | 清空整个场景的压力场（所有cell的Pressure归零，其余字段不变）|

关于画笔预览与提交的说明：这是一个明确的两阶段交互——拖拽过程中看到的是"即将绘制"的预览效果，只有松开鼠标才真正影响仿真。如果这一交互设计存在歧义或实现困难，请按此理解实现，并在交付时说明你的具体处理方式。
